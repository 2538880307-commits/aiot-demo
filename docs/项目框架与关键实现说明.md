# 轨道交通检修工具监测平台：项目框架与关键实现说明（当前版本）

## 1. 项目目标与边界

本系统面向轨道交通检修业务，聚焦四类能力：

1. 工具台账管理：统一维护工具基础信息与库存。
2. 工具识别对比：基于 YOLO 对检修前后图片进行数量对比。
3. 权限治理：管理员分配权限，员工按授权访问页面与接口。
4. 审计追踪：关键业务操作写入日志并支持检索。

## 2. 总体架构

系统采用前后端分离 + 容器编排：

1. 前端：Vue3 + Vite + Element Plus。
2. 后端：FastAPI + SQLAlchemy(Async) + Pydantic。
3. 数据与中间件：PostgreSQL、Redis、EMQX。
4. AI 能力：YOLO 模型文件通过卷挂载到后端容器。

## 4. 关键业务流程

### 4.1 登录与会话

1. 前端调用 `POST /api/v1/auth/login`。
2. 后端校验 `username/password`（哈希密码）后返回角色和权限集合。
3. 前端将会话写入 `localStorage`，路由守卫按权限放行。

### 4.2 工具管理

1. 前端在“工具管理”页调用 `/api/v1/tools` 系列接口。
2. 后端按 `requester_username` 做权限校验（管理员或具备“工具管理”权限）。
3. 对工具编码唯一性、工具类型有效性做校验后入库。
4. 增删改操作同步写入 `operation_logs`。

### 4.3 工具识别

1. 前端“工具识别”页分别上传“检修前/检修后”图片。
2. 后端接口 `POST /api/v1/tool-count/detect` 调用 `tool_count_service`。
3. 返回 `total_count`、`by_class`、`detections` 等识别结果。
4. 前端计算差异并给出“缺失/一致/新增”结论。

### 4.4 系统设置与审计

1. 管理员可维护密码策略、告警阈值、工具类型字典。
2. 所有关键写操作通过 `append_log` 写入操作日志。
3. 管理员可通过 `/api/v1/settings/operation-logs` 分页检索。

## 5. 数据模型（核心）

1. `tools`：工具编码、类型、名称、库存、维护组、图片地址。
2. `users`：账号、工号、姓名、部门、职位、角色、权限、密码哈希。
3. `tool_types`：工具类型字典（名称、描述、排序、启用状态）。
4. `system_settings`：系统参数（JSON 值）。
5. `operation_logs`：模块、动作、操作者、目标、详情、时间。

## 6. 后端实现要点

### 6.1 启动生命周期

`backend/app/main.py` 在应用启动时完成：

- 自动建表（`Base.metadata.create_all`）
- 兼容性补丁（`users.password_hash` 列）
- 种子初始化（工具、用户、系统配置）
- MQTT 服务启动

### 6.2 权限控制

`backend/app/api/routes.py` 使用：

- `require_admin`：管理员限定接口
- `require_permission`：按业务权限校验（如“工具管理”“工具识别”）

权限控制在后端生效，不依赖前端显示隐藏。

### 6.3 识别服务

`backend/app/services/tool_count_service.py` 负责模型加载与推理；模型路径来自 `TOOL_COUNT_MODEL_PATH`，默认容器路径 `/app/model/best.pt`。

### 6.4 审计日志

`append_log` 在工具管理、权限管理、系统设置、工具识别成功/失败场景均有记录。

## 7. 前端实现要点

### 7.1 路由与权限

`frontend/src/router/index.js` 为每个业务路由声明 `meta.permission`：

- `工具管理` -> `/tool-recognition`
- `工具识别` -> `/tool-count`
- `权限管理` -> `/permission-management`
- `系统设置` -> `/system-settings`

### 7.2 页面模块

1. `ToolRecognitionView.vue`：工具台账管理。
2. `ToolCountView.vue`：识别对比、分类差异、识别日志（管理员可见）。
3. `PermissionManagementView.vue`：用户维护与权限分配。
4. `SystemSettingsView.vue`：策略与字典配置、日志查询。

## 9. 当前版本限制

1. 鉴权仍基于会话 + `requester_username` 参数，未引入 JWT。
2. WebSocket 告警通道仅保留基础连接能力。
3. 识别结果当前主要用于页面展示和日志，未做完整业务闭环。
